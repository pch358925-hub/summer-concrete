# 서중 콘크리트 습윤양생 사진대지

카카오톡, 모바일 브라우저, PC에서 1일차부터 5일차까지 습윤양생 사진을 등록하고 A4 사진대지로 출력하는 웹앱입니다.

현재 GitHub Pages 주소:

```text
https://pch358925-hub.github.io/how-weather-concrete/
```

## 배포 파일

GitHub Pages 실행에 필요한 파일은 아래 4개입니다.

- `index.html`
- `styles.css`
- `app.js`
- `config.js`

`README.md`는 저장소 설명용 파일이라 앱 실행에는 필요하지 않지만, 현재 상태 기록용으로 같이 보관합니다.

## 주요 기능

- 사진대지 생성 및 실시간 공유 저장
- 현장명, 타설부위, 타설일 입력
- 작업월 기준 사진대지 목록 조회 및 타설부위 검색
- 1일차부터 5일차까지 사진 촬영 또는 첨부
- 첨부에서 여러 장 선택 시 시작 일차부터 순서대로 자동 등록
- 사진 업로드 전 현장 공유용 JPG 자동 압축
- HEIC/HEIF 사진은 JPG 변환 시도
- 사진 삭제 시 화면 즉시 반영
- 재접속 시 목록 개수와 상세 사진 데이터 보정
- 등록된 사진 클릭 시 확대 보기
- 확대 보기에서 배경/여백 터치, 닫기 버튼, Esc로 닫기
- A4 사진대지 출력 및 PDF 저장
- 저장 용량 게이지 표시

## 사용 방법

1. `새 대지`를 눌러 사진대지를 만듭니다.
2. 현장명, 타설부위, 타설일을 입력합니다.
3. 필요한 사람에게 현재 페이지 주소를 공유합니다.
4. 각 일차 카드에서 `촬영` 또는 `첨부`로 사진을 등록합니다.
5. 사진을 누르면 크게 볼 수 있습니다.
6. 등록이 끝나면 일반 브라우저에서 `인쇄/PDF`를 눌러 출력합니다.

## 장기 사용 메모

- 공유 주소는 하나 그대로 사용합니다.
- 사진대지 목록은 선택한 `작업월` 자료만 불러와서, 몇 달치 사진이 쌓여도 첫 화면이 무겁지 않게 관리합니다.
- 이전 자료는 작업월의 이전/다음 버튼이나 월 선택으로 확인합니다.
- 용량 게이지는 저장소 전체 사진 개수를 기준으로 대략 계산합니다.
- 새로 등록하는 사진은 긴 변 기준 최대 1440px, 목표 520KB 안팎으로 자동 압축합니다.

## 모바일 사용 메모

- 카카오톡 안에서는 첨부가 한 장만 선택되는 경우가 있습니다.
- 여러 장 첨부는 갤럭시 인터넷 또는 크롬에서 더 안정적입니다.
- 카카오톡 안에서는 `인쇄/PDF`가 막힐 수 있으므로 출력은 일반 브라우저에서 진행합니다.
- 같은 사진대지를 PC, 모바일, 카톡에 동시에 열어둔 상태에서 사진을 첨부하면 실시간 갱신이 겹칠 수 있습니다. 현재는 첨부/저장 중 갱신을 미루도록 보정되어 있지만, 테스트할 때는 한 창만 열고 확인하는 것이 가장 안정적입니다.

## 사진 등록 동작

- `1일차 첨부`에서 5장을 선택하면 1일차부터 5일차까지 순서대로 등록됩니다.
- `2일차 첨부`에서 여러 장을 선택하면 2일차부터 5일차까지 순서대로 등록됩니다.
- 5일차를 넘는 사진은 제외됩니다.
- 처리 실패한 사진은 건너뛰고 다음 사진으로 이어서 채웁니다.
- 기존 사진이 있는 칸에 여러 장을 등록하면 덮어쓰기 전에 확인창이 뜹니다.

## GitHub Pages 배포

1. GitHub 저장소에 `index.html`, `styles.css`, `app.js`, `config.js`를 올립니다.
2. 저장소 `Settings > Pages`에서 `Deploy from a branch`를 선택합니다.
3. 브랜치는 `main`, 폴더는 `/root`로 둡니다.
4. 배포가 끝나면 아래 주소로 접속합니다.

```text
https://pch358925-hub.github.io/how-weather-concrete/
```

## Supabase 설정

`config.js`에 Supabase 연결값이 들어 있어야 여러 사람이 같은 사진대지를 실시간으로 공유할 수 있습니다.

```js
window.CONCRETE_PHOTO_CONFIG = {
  supabaseUrl: "https://YOUR_PROJECT.supabase.co",
  supabaseAnonKey: "YOUR_SUPABASE_ANON_KEY",
  bucket: "curing-photos",
};
```

## 관리 메모

- 새 사진대지는 `새 대지` 버튼을 눌렀을 때만 생성됩니다.
- 삭제한 사진대지는 자동으로 다시 생성되지 않습니다.
- 사진대지 목록은 날짜와 생성일 기준으로 정렬됩니다.
- 출력 제목과 표는 A4 세로 사진대지 기준으로 맞춰져 있습니다.
