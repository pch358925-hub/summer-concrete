# 서중 콘크리트 습윤양생 사진대지

카카오톡으로 링크를 공유하고, 모바일 또는 PC에서 1일차부터 5일차까지 습윤양생 사진을 등록한 뒤 A4 사진대지로 출력하는 웹앱입니다.

## 배포 파일

GitHub Pages 실행에 필요한 필수 파일은 아래 4개입니다.

- `index.html`
- `styles.css`
- `app.js`
- `config.js`

`README.md`는 저장소 설명용 파일이라 있어도 되고 없어도 됩니다. 앱 실행에는 영향을 주지 않습니다.

## 주요 기능

- 사진대지별 공유 링크 생성
- 현장명, 타설부위, 타설일 입력
- 1일차부터 5일차까지 사진 촬영 또는 첨부
- 사진 업로드 전 자동 압축
- 사진대지 목록에서 등록 현황 확인
- A4 사진대지 출력 및 PDF 저장
- 용량 게이지 표시

## 사용 방법

1. `새 대지`를 눌러 사진대지를 만듭니다.
2. 타설부위와 타설일을 입력합니다.
3. `링크` 버튼으로 주소를 복사해 카톡방에 공유합니다.
4. 현장 담당자가 `촬영` 또는 `첨부`로 1일차부터 5일차 사진을 등록합니다.
5. 등록이 끝나면 브라우저에서 `인쇄/PDF`를 눌러 출력합니다.

## 카카오톡 사용 주의

카카오톡 안에서 열린 화면은 사진 등록과 확인은 가능하지만, 카톡 WebView 특성상 `인쇄/PDF`가 막힐 수 있습니다.

출력할 때는 카톡 화면에서 브라우저로 열거나, 삼성 인터넷/크롬 같은 일반 브라우저에서 링크를 열어 `인쇄/PDF`를 사용하세요.

## 날짜 기준

- `타설일`은 사진대지의 실제 타설 날짜입니다.
- 1일차부터 5일차 날짜는 타설일을 기준으로 자동 계산됩니다.
- 사진대지 목록에는 전체 사진대지가 표시됩니다.

## GitHub Pages 배포

1. GitHub 저장소에 필수 파일 4개를 올립니다.
2. 저장소 `Settings > Pages`로 이동합니다.
3. `Deploy from a branch`를 선택합니다.
4. 브랜치는 `main`, 폴더는 `/root`를 선택합니다.
5. 발급된 `https://계정명.github.io/저장소명/` 주소를 엽니다.

## Supabase 설정

`config.js`에 Supabase 연결값이 들어 있어야 여러 사람이 같은 사진대지를 실시간으로 공유할 수 있습니다.

```js
window.CONCRETE_PHOTO_CONFIG = {
  supabaseUrl: "https://YOUR_PROJECT.supabase.co",
  supabaseAnonKey: "YOUR_SUPABASE_ANON_KEY",
  bucket: "curing-photos",
};
```

## 사진 용량

사진은 업로드 전에 브라우저에서 자동으로 줄여 저장합니다. 용량 게이지는 등록된 사진 수를 기준으로 대략적인 사용량을 표시합니다.

## 관리 메모

- 새 사진대지는 `새 대지` 버튼을 눌렀을 때만 생성됩니다.
- 삭제한 사진대지는 자동으로 다시 생성되지 않습니다.
- 출력 제목과 표는 A4 사진대지 기준으로 맞춰져 있습니다.
